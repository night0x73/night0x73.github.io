Vehicle Inventory Management System MVC

Background:
After graduating college, I started to relearn Python and Java. This project started as a Python app that ran through the IDE console and stored data in an excel spreadsheet. My desire to learn backend development and proficiency in Java led me to changing the language this app was written in. After learning about the MVC design pattern, I was able to produce the model and view components in Java, but my grasp on Controllers (HttpSerlvet specicially) wasn't so great, so I decided to learn Spring in order to make my application more robust and modular. This is by far the largest software project I've worked on as a programmer, but I hope to grow and eventually use this knowledge in the industry.

Abstract:
To create a program that manages vehicle inventory data for a car dealership. This project teaches me about Java development at the enterprise level by implementing technologies offered by the Spring project. I am focusing on the Model-View-Controller design pattern with interactions between three main entities (Vehicle, CustomerAccount, FinanceRecords). As entities are added to this project, especially ones with relationships with existing ones, complexity grows at a nonlinear rate. Due to the benefits of dependency injection and the annotation-based configuration magic offered by Spring, users of this application have free control over data input.

Tools/Technologies:

  • Build tools: Maven
  • Frameworks: Spring Core, Hibernate
  • Database: MySQL + MySQLWorkbench, JDBC
  • Server: Apache Tomcat
  • View layer: HTML/CSS + JSP, JSTL

I've spent a fair amount of time this past week understanding how to further use Hibernate mappings to create relationships between my tables. Current schema setup: 3 tables: Cars, CustomerAccounts, FinanceRecords Each of these tables are connected to the backend by a DAO layer, and all three of these DAOs pass through a service layer called VehicleSerivce.class Foreign key setup: financeId in Vehicle is the FK to financeId PK in FinanceRecord, customerId in financeRecord is FK to customerId PK in CustomerAccount. Vehicle object holds a one-to-one bidirectional relationship with FinanceRecord. Vehicle is the owner object, a FinanceRecord needs a Vehicle to exist. FinanceRecord holds a many-to-one relationship with CustomerAccount, as one CustomerAccount can finance multiple Vehicles. Therefore, a CustomerAccount holds a OneToMany relationship with FinanceRecord. Stock Screener REST

Background/Abstract:
To create a stock screening program that takes advantage of automation to notify the user in real time when a stock hits a certain price.

Background:
As a hobbyist trader myself, I got tired of constantly checking my watchlist. This program seeks to solve that problem. Now, I'm neither a frequent trader, nor do I invest in quantities a seasoned trader would. The simplicity of this program supports the aforementioned notion. On to the process. Initially, a public API such as Alpha Vantage or Yahoo_fin is used to retrieve real-time stock prices. This data is processed by a Python script, which will feed into the database. As long as the scheduled Cron task is active, the system will periodically retrieve prices at set time intervals. Having stored the data in MySQL, Spring Boot is used to build a REST service that will feed into an in-browser stock screener.

Tools/Technologies:
  • Eclipse IDE + java.net.http + java dateTime
  • Spring Boot + Maven
  • Python
  • MySQL + MySQL Workbench
  • Jackson (for JSON)
  • Alpha Vantage/Yahoo_fin API
  • Excel
Notes:
It could be a good idea to create some sort of watch list using a data structure and looping through the watch list at set times. Most real-time financial data requires a paid-service, but Alpha Vantage is free and the functionality is enough for the purposes of this project. I am not expecting the Alpha Vantage API to retrieve data with accuracy of under a minute.

Phase 1:
Establishing a connection to the Alpha Vantage API/Yahoo_fin and being able to grab the stock price of any ticker symbol.

New things I learned:
  • Parsing JSON files and accessing stock data through JSON file structure.
  • API requests: most common are GET, POST, PATCH, PUT, DELETE
  • Object serialization/deserialization
  • Database: MySQL + MySQLWorkbench, JDBC
  • Server: Apache Tomcat
  • Maven and XML
Idea: Mapping timestamps to their respective data in the database.

Phase 2: Using SQL qeuries within the Python script to store data in the database.

Phase 3: Creating a entity model for a stock ticker. Employing the MVC pattern in Spring Boot and creating the REST service.

Phase 4: Feed REST service into frontend.

Solar Panel Optimization

Background:
Alternative/renewable energy has always been something I cared about; especially since it can benefit our environment and natural resources on this earth are finite. Since the sun is so prevalent in our everyday lives, solar energy seems like a good place to start since it's so abundant. There are many impediments that reduce the energy that is captured by photovoltaic cells. One of my tasks is to understand which of those roadblocks can be eliminated and which will inevitably affect photon availability. I hypothesize that the angle of the surface area of solar cell placement will greatly affect the energy absorbed. Therefore, I want to build a circuit with a motor that can change the angle of the panel such that it will absorb the most energy. Detection of light is done via photoresistor sensors. If this application is successful for a single small solar panel, it's size can be scaled down and woven into a flexible solar panel 'blanket'.

Abstract:
Can I build a home-made circuit such that it optimizes the energy that solar panels can retain by automating the movement of solar cells?

Physics/laws of nature to understand:
  • Earth's atmosphere is made of 5 layers, how to photons interact with each layer? Due to the physical properties of light, energy can be lost as it passes through our atmosphere.
Questions and challenges:
  • How do sensors measure the intensity of the ray?
  • Is this feasible for how much energy is required to move the motor? If the energy coming in doesn't exceed the energy used to rotate the panels, then this application is impractical. Follow-up: How can I can minimize the amount of electricity required to run the motors? Is it possible to use some of the energy that is captured?
